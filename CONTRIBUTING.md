# Contributing

Found a new endpoint? Found a bug in the docs?

## How to contribute

1. Fork the repository
2. Create a branch: `git checkout -b add/new-endpoint`
3. Add or update the relevant doc under `docs/`
4. Add a real request + real response example
5. Verify it yourself and label it (VERIFIED / PARTIALLY VERIFIED / UNVERIFIED)
6. Open a pull request

## Documentation style

Match the existing style exactly:
- Endpoint URL first
- Short description
- Real curl example
- Trimmed real JSON response
- Status label at the bottom

No essays. No speculation. Only real, testable endpoints.
